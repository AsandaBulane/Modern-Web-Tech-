HTML Basics

HTML is the foundation of web development.

It defines the structure of web pages using markup.

Tags are used to delineate elements like headings, paragraphs, links, etc.

HTML5 introduced new semantic elements like <article> and <section>.

Accessibility begins with semantic HTML.

  • <h1> to <h6> for headings
  • <p> for paragraphs
  • <a> for links
  • <div> for divisions
  • <span> for inline elements
<h1>Hello World</h1>
CSS Fundamentals

CSS styles the structure defined by HTML.

CSS3 added powerful features like Flexbox and Grid.

Styles are defined in rules with selectors and declarations.

You can apply styles inline, in the head, or in external stylesheets.

Classes and IDs help target specific elements.

  • Selectors
  • Properties
  • Box model
  • Display types
  • Responsive design
body { font-family: Arial; }
JavaScript Essentials

JavaScript brings interactivity to web pages.

It runs in the browser and can manipulate the DOM.

Modern JS supports ES6+ features like arrow functions and classes.

It is event-driven and asynchronous in nature.

APIs like Fetch enable network requests directly from the browser.

  • Variables
  • Functions
  • DOM manipulation
  • Events
  • Async/await
console.log("Hello, JS!");
Responsive Design

Responsive design ensures your site looks good on all devices.

Media queries are a powerful CSS feature for this.

Flexbox and Grid help layout elements in adaptive ways.

Mobile-first design is the recommended approach today.

Viewport meta tag is essential for responsiveness.

  • Flexible layouts
  • Relative units
  • Breakpoints
  • Accessibility
  • Performance
@media (max-width: 600px) { nav { display: none; } }
APIs and Fetch

APIs let your app interact with external data and services.

RESTful APIs return JSON data typically.

Fetch API is built into modern browsers.

You can chain promises to handle asynchronous operations.

Error handling is crucial with network requests.

  • GET and POST requests
  • Status codes
  • Headers
  • Authorization
  • Error handling
fetch('https://api.example.com/data')
